# 🍜 텍스트 분석 기반 레스토랑 추천 시스템 (KoBART & SBERT 활용)

## 📌 프로젝트 개요

본 프로젝트는 네이버 리뷰 데이터를 활용하여 사용자의 구체적인 요구사항과 선호하는 분위기를 반영한 **개인 맞춤형 레스토랑 추천 시스템**을 개발합니다.

단순히 평점이나 키워드 빈도에 의존하는 기존 방식의 한계를 극복하고, **자연어 처리(TA)** 기술을 활용하여 리뷰 텍스트의 문맥과 의미를 깊이 있게 분석하여 정교한 추천 결과를 제공합니다.

---

## ⚙️ 주요 기술 스택 및 데이터 처리 파이프라인

### 1. 리뷰 데이터 요약 (KoBART 기반)
* **KoBART (Korean BART)**: 대규모 네이버 리뷰 데이터의 노이즈와 편향성을 줄이기 위해, 한국어에 최적화된 KoBART 모델을 사용하여 리뷰들을 **추상적 요약(Abstractive Summarization)**했습니다.
* **재귀적 요약**: 리뷰들을 토큰 단위로 그룹화하고 요약하는 과정을 반복하여, 한 식당당 하나의 포괄적이면서도 균형 잡힌 최종 요약본을 생성했습니다.

### 2. 추천 엔진 기술 (SBERT & Cosine Similarity)
* **Sentence-BERT (SBERT)**: 사용자 입력 문장과 요약된 리뷰 텍스트의 **문맥적 의미**를 정확하게 파악하고 벡터로 변환합니다.
* **Cosine Similarity**: 이 벡터들을 비교하여 텍스트 간의 의미적 유사성(리뷰 유사성)을 수치화하고, 추천 점수의 핵심 기준으로 활용합니다.

---

## 🌟 추천 모델 구성 및 특징 (2가지 전략)

사용자의 다양한 요구를 충족시키기 위해 두 가지 추천 모델을 설계했으며, 각 모델은 다른 추천 전략을 가집니다.

### Model 1: 리뷰 유사성 집중 모델 (Review Similarity Focused)
* **핵심 기준**: 요약 리뷰와 사용자 입력 간의 **텍스트 유사성**에 가장 높은 가중치를 부여합니다.
* **활용 요소**: 사용자 입력에서 지역 및 카테고리 키워드를 추출하여 1차 필터링을 수행하고, 동반자 정보는 보조적인 가중치로만 반영됩니다.
* **특징**: 사용자 요청의 **분위기**나 **음식 설명** 등 텍스트 기반의 요구사항을 빠르고 효과적으로 반영합니다.

### Model 2: 다중 요소 균형 반영 모델 (Multi-Factor Balanced)
* **핵심 기준**: 텍스트 유사성, **대기 시간(Wait Time) 정보**, **동반자 비율** 등 세 가지 주요 요소를 균형 있게 결합하여 최종 점수를 산출합니다.
* **활용 요소**: 사용자에게 누락된 정보를 질문하여 수집하고, 각 요소의 중요도에 따라 가중치를 조정하여 점수에 반영합니다.
* **특징**: 실제 레스토랑 선택 시 고려되는 현실적인 요소들을 통합하여 **가장 개인화되고 정교한** 추천 결과를 제공합니다.
